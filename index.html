Understanding Avidity's Scale & Multi-Agency Structure

With 1,240 employees, £65M turnover, and 16% year-on-year growth across five specialist agencies (McCurrach, Standout, Wave, Thumbprint, and Zing), Avidity operates at significant scale in the fast-moving FMCG sector. My approach will ensure digital practices can support this continued growth while maintaining operational excellence across all agencies.

Strategy

Purpose: Design the future state, set direction, and plan improvements for engineering and systems.

Software Engineering Excellence

  • Technical Roadmap: Develop a comprehensive technical roadmap that balances immediate business needs with long-term architectural goals, focusing on the modernisation of legacy systems.
  • Engineering Best Practices: Implement SOLID principles, domain-driven design, and clean architecture practices across all new development while gradually refactoring legacy code.
  • Code Quality: Establish automated code quality gates, comprehensive code reviews, and pair programming sessions to ensure consistent quality standards.
  • Architecture Governance: Create an architecture review board for critical system decisions, ensuring alignment with business strategy and technical standards.
  • Dependency Management: Leverage GitHub's Dependabot and similar tools to automate dependency updates, ensuring security patches are applied promptly and reducing technical debt.
  • Comprehensive Testing Strategy: Implement multi-layered testing including unit, integration, end-to-end, performance, security, and chaos engineering.
  • Developer Experience: Create efficient development environments and tooling that reduce friction and accelerate delivery while maintaining quality standards.

System Integration & Platform Strategy

  • API Strategy: Develop a comprehensive API strategy with standardised patterns for RESTful services, including versioning, documentation, and monitoring.
  • Salesforce & Power Platform Integration: Create robust integration patterns between custom applications and Avidity's Salesforce, SharePoint, and Power Platform implementations.
  • Service Contracts: Implement formal service contracts and API governance to ensure consistency and reliability across all integrations.
  • Data Consistency: Establish data standardisation patterns and conflict resolution strategies to maintain consistency across distributed systems.

DevOps & Delivery Excellence

  • CI/CD Implementation: Build modern CI/CD pipelines using GitHub Actions and Azure DevOps with comprehensive automated testing.
  • Deployment Automation: Implement zero-downtime deployments, automated rollback capabilities, and deployment approval workflows.
  • Performance Metrics: Track and improve key metrics including deployment frequency, lead time, change failure rate, and MTTR.
  • Environment Strategy: Establish consistent dev/test/staging/production environments with infrastructure-as-code for reproducibility.

Environment Strategy & Infrastructure

  • Environment Standardisation: Implement clearly defined, consistent environments with specific purposes (local, dev, QA, staging, production).
  • Specialised Environments: Purpose-built environments for AI/ML, data science, and security testing.
  • Environment Parity: Ensure consistency across environments through IaC, configuration-as-code, and container orchestration.
  • Environment Governance: Implement controls for environment management, access, cost, and compliance.

Azure Cloud-Native Architecture

  • Cloud Migration: Lead the migration from on-premises or legacy hosting to Azure PaaS services using a phased approach.
  • Serverless Architecture: Implement Azure Functions and Logic Apps for event-driven processing of EPOS data and field reports.
  • Containerisation: Adopt container-based deployments for consistent environments and efficient resource utilisation.
  • Cost optimisation: Implement cloud cost monitoring and optimisation strategies, including auto-scaling and right-sizing of resources.
  • Azure Resource Manager Templates: Standardise infrastructure deployment using ARM templates for consistent, repeatable environment creation.
  • Azure Policy: Implement governance guardrails with custom and built-in policies to enforce organisational standards and compliance requirements.
  • Azure Cost Management: Utilise Cost Analysis tools to monitor spending patterns, implement budget alerts, and optimise resource allocation across environments.
  • Azure Monitor & Application Insights: Deploy comprehensive monitoring for infrastructure health, application performance, and user experience metrics.

Azure Governance & Cost Management

  • Azure Blueprints: Create standardised environment templates that combine Resource Manager templates, role assignments, and policy assignments for consistent deployment across subscriptions.
  • Management Groups: Implement a hierarchical structure for organising subscriptions and applying governance controls at scale across the organisation.
  • Azure Policy as Code: Manage policies through version-controlled configuration files, enabling automated policy deployment and compliance reporting.
  • Resource Tagging Strategy: Develop a comprehensive tagging taxonomy for cost allocation, ownership tracking, and environment classification.
  • Cost optimisation Framework: Implement a structured approach to cost management including reserved instances, auto-shutdown, right-sizing, and anomaly detection.
  • Azure Advisor: Leverage recommendations for high availability, security, performance, and cost optimisation across all Azure resources.
  • Governance Dashboards: Create custom Power BI dashboards for compliance status, cost trends, and resource utilisation across the organisation.

Data Engineering & Intelligence

  • Data Standardisation: Implement common data formats, schemas, and taxonomies across all agencies to ensure consistency and interoperability.
  • Database Strategy: Optimise database usage across MSSQL, PostgreSQL, and MySQL/MariaDB for efficient data storage, retrieval, and analysis.
  • Data Pipelines: Implement efficient ETL processes using Azure Data Factory for EPOS data integration and analysis.
  • Real-time Analytics: Build real-time data processing capabilities for immediate insights from field operations and sales data.
  • Machine Learning: Develop predictive models for sales forecasting, inventory optimisation, and customer behavior analysis.

Security & Governance

  • Secure SDLC: Implement security throughout the development lifecycle, including threat modeling and security code reviews.
  • Authentication & Authorisation: Standardise on modern authentication protocols and implement proper role-based access control across all systems.
  • Compliance: Ensure all systems meet GDPR, ISO 27001, and industry-specific compliance requirements.
  • Security Automation: Implement automated security scanning in CI/CD pipelines and regular vulnerability assessments.
  • Azure Key Vault: Centralise secret management using Azure Key Vault for secure storage of API keys, certificates, and connection strings.
  • Azure Security Center: Deploy advanced threat protection with Security Center for continuous security assessment and actionable recommendations.
  • Azure Sentinel: Implement SIEM capabilities for security information and event management across all Avidity systems.
  • Azure Private Link: Secure data transfers with private endpoints to eliminate data exposure on the public internet.
  • Azure Information Protection: Implement data classification and protection policies to secure sensitive information across the organisation.

Modern Practices & Tools

  • Foster a curious, supportive environment for open discussion and continuous improvement.
  • Introduce modern AI tools and platforms such as Claude, GitHub Copilot, Cursor, MCP servers, n8n, Goose, Power Automate, and Ollama (local LLMs).
